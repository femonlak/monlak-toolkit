# Pipeline Context - Linear Structure

## Time: Partners

Sub-team de Pillars. Status customizados para funil comercial.

## Projetos (Funis)

| Projeto | Descrição |
|---------|-----------|
| Poker Operators | Sites de poker online |
| Offline Cashier | Clubes, torneios, poker presencial |
| Affiliate Partners | Programa de afiliados (todos os tiers) |
| Payment Integrations | Processadores além de poker (Forex, etc) |

## Pipeline Status (ordem do funil)

| Status | Categoria Linear | Descrição |
|--------|-----------------|-----------|
| Backlog | Backlog | Prospects identificados, não priorizados. FORA do pipeline ativo. |
| To Contact | Unstarted | Priorizados e validados como fit, aguardando primeiro contato. |
| Contacted | Started | Primeiro contato feito, aguardando resposta. |
| In Conversation | Started | Conversas ativas, trocando informações e apresentando proposta. |
| Negotiating | Started | Negociação comercial ativa (comissões, SLAs, exclusividade). |
| Contract Review | Started | Termos acordados, contrato em análise jurídica. |
| Integration | Started | Contrato assinado, integração técnica em andamento. |
| In Discussion | Started | Parceiro Live com conversa ativa (renegociação, expansão, novo produto). |
| Live | Completed | Operando em produção, processando transações. |
| Lost | Canceled | Deal não aconteceu (não respondeu, concorrente, timing, fit ruim). |

## Status Ativos para Scoring

Universo de análise do Pipeline Activity Score: **To Contact a In Discussion** (7 status).

Backlog, Live e Lost ficam fora do scoring de atividade.

## Mecânica de Operação

- **Issues** = oportunidades/parceiros
- **Comentários** = atualizações de cada oportunidade (histórico de contatos, reuniões, decisões)
- **Status** = posição no funil
- **Prioridade** = importância estratégica da oportunidade
- **Project Updates** = visão consolidada do funil para reporting

## Formato de Comentário Padrão

Cada atualização de oportunidade segue este formato:

```markdown
**[Contexto do encontro/contato]**

**Situação:**
- [pontos principais do que aconteceu]

**Próximos passos:**
- [ações concretas com responsável quando possível]

**TL;DR:** [1 linha resumindo status e próximo passo crítico]
```
