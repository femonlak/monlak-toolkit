# UX/UI Guidelines

Traduzir requisitos de produto em diretrizes práticas de interface e experiência.

## Análise Prévia
Ler do PRD:
- User journey completo
- Critérios de aceitação
- Edge cases

## Estrutura das Diretrizes

### Fluxo de Telas
- Navegação e estrutura
- Transições entre estados
- Hierarquia de informação

### Componentes Principais
- Elementos-chave da interface
- Campos, botões, cards
- Agrupamento lógico

### Hierarquia Visual
- Priorização de elementos
- Estrutura de layout
- Espaçamento

### Feedback e Estados
- Loading states
- Mensagens de erro/sucesso
- Validações em tempo real
- Estados vazios

### Micro-interações
- Animações e transições
- Hover e focus states
- Feedback visual de ações

### Acessibilidade
- Navegação por teclado
- Labels e ARIA
- Contraste e legibilidade
- Foco visível

## Processo

1. Analisar PRD
2. Montar proposta completa
3. Apresentar: "Criei diretrizes de UX/UI. O que você achou?"
4. Iterar até aprovação
5. Ao aprovar: "Posso incorporar no documento?"
6. Adicionar seção `## UX/UI GUIDELINES` ao PRD

## Princípio
Proporcionalidade: features simples = diretrizes enxutas.

## Nunca Fazer
- Incluir detalhes técnicos de implementação
- Criar documentos enormes com info desnecessária
- Forçar detalhes irrelevantes para a feature
- Incorporar sem aprovação explícita
