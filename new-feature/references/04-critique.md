# Critique

Revisar documento através de perguntas estruturadas para identificar lacunas.

## Processo

### 1. Identificar Objetivo
- Se explícito no documento, usar
- Se não, inferir do conteúdo

### 2. Análise Silenciosa
Identificar (2-5 pontos):
- Informações vagas ou ambíguas
- Cenários não contemplados
- Inconsistências entre seções
- Ângulos importantes não capturados

**Foco:** Apenas o que falta para completar o documento. Evitar over-analysis.

### 3. Abertura
```
Entendi que o objetivo do documento é [objetivo]. Tenho [N] questionamentos para melhorá-lo. Podemos iniciar?
```
Aguardar confirmação.

### 4. Entrevista
UMA pergunta por vez:
```
## Pergunta [X] de [Y]

[Contexto/lacuna - 1-2 frases]

[Pergunta objetiva]

| # | Opção |
|---|-------|
| 1 | [Alternativa] |
| 2 | [Alternativa] |
| 3 | [Alternativa] |

**Qual opção prefere, ou descreva o que quer.**
```

Após resposta: "Entendido. [resumo]" → próxima pergunta.

### 5. Atualização
Para cada resposta:
- Melhora seção existente → editar diretamente
- Contexto novo → adicionar em `## INFORMAÇÕES ADICIONAIS`

## Quando NÃO Perguntar
Se documento está completo e claro:
```
Revisei o conteúdo e está completo. Não identifiquei lacunas que precisem de esclarecimento.
```

## Nunca Fazer
- Múltiplas perguntas simultâneas
- Expandir escopo original
- Forçar perguntas se documento está completo
- Perguntar sobre implementação técnica
- Adicionar info que não veio das respostas
