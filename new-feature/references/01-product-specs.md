# Product Specs

Estruturar PRD focado em user value e requisitos de negócio.

## Estrutura do PRD

### 1. Nome objetivo
Formato: Verbo + Substantivo (ex: "Exportar Relatório")

### 2. Descrição curta
"[Feature] é [tipo] que [função]." Máx 20 palavras.

### 3. Problema
Dor específica com dados concretos. Ex: "usuários gastam 2h/dia fazendo X manualmente"

### 4. Job to be Done
"Quando [situação], quero [tarefa], para [resultado]."

### 5. User Story
"Como [usuário], quero [ação], para [resultado]."

### 6. Solução
Por que a feature elimina/reduz o problema. Foco no benefício.

### 7. User Journey
Passos sequenciais. Cada passo = 1 verbo + 1 substantivo.
1. [Verbo] [substantivo]
2. [Verbo] [substantivo]
...

### 8. Critérios de Aceitação
"Dado [contexto], quando [ação], então [resultado]."

### 9. Edge Cases
"Se [condição rara], então [comportamento esperado]."

## Processo

1. Analisar input do usuário
2. Definir nome objetivo (verbo + substantivo)
3. Criar arquivo PRD
4. Preencher estrutura
5. Apresentar ao usuário
6. Iterar até aprovação
