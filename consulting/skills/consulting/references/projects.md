# Project & Change Management Frameworks

Frameworks for planning projects, defining roles, and driving organizational change.

## Work Breakdown Structure (WBS)

**Purpose**: Decompose a project into manageable components for planning and tracking.

### Structure

```
Project Goal
├── Phase 1 / Deliverable 1
│   ├── Work Package 1.1
│   │   ├── Task 1.1.1
│   │   └── Task 1.1.2
│   └── Work Package 1.2
├── Phase 2 / Deliverable 2
│   ├── Work Package 2.1
│   └── Work Package 2.2
└── Phase 3 / Deliverable 3
```

### WBS Principles

1. **100% Rule**: WBS must include 100% of the work - nothing more, nothing less
2. **Mutually Exclusive**: No overlap between elements at the same level
3. **Outcome-Focused**: Define deliverables, not activities
4. **Appropriate Depth**: Decompose until work packages are estimable (8-80 hour rule)

### How to Create a WBS

1. **Start with the end goal** - What is the final deliverable?
2. **Identify major phases or deliverables** - What are the main components?
3. **Decompose each phase** - Break into smaller deliverables
4. **Continue decomposing** - Until work packages are manageable
5. **Assign IDs** - Create a numbering scheme (1.0, 1.1, 1.1.1)
6. **Validate** - Check against 100% rule

### WBS Dictionary Entry

| Field | Description |
|-------|-------------|
| WBS ID | Unique identifier (e.g., 1.2.3) |
| Name | Work package name |
| Description | What this work package delivers |
| Owner | Responsible person |
| Duration | Estimated time |
| Dependencies | Predecessor work packages |
| Deliverables | Specific outputs |
| Acceptance Criteria | How to verify completion |

## RACI Matrix

**Purpose**: Define roles and responsibilities for each task or decision.

### The RACI Roles

| Role | Definition | Rules |
|------|------------|-------|
| **R** - Responsible | Does the work | At least one per task |
| **A** - Accountable | Has final authority, approves | Exactly one per task |
| **C** - Consulted | Provides input (two-way) | As needed |
| **I** - Informed | Kept updated (one-way) | As needed |

### RACI Matrix Template

| Task / Decision | Person A | Person B | Person C | Person D |
|-----------------|----------|----------|----------|----------|
| Task 1 | A | R | C | I |
| Task 2 | I | A | R | C |
| Task 3 | C | I | A | R |
| Decision 1 | A | C | C | I |

### RACI Best Practices

1. **One A per row** - Only one person can be accountable
2. **Limit C's** - Too many consulted slows decisions
3. **A can also be R** - Accountable person can also do the work
4. **Validate with stakeholders** - Everyone should agree on their role
5. **Review regularly** - Update as project evolves

### Common RACI Problems

| Problem | Symptom | Solution |
|---------|---------|----------|
| No A | Tasks fall through cracks | Assign accountability |
| Multiple A's | Confusion, conflict | Clarify single owner |
| Too many R's | Unclear who does what | Define primary R |
| All C's | Slow decisions | Reduce consultations |

## Kotter's 8-Step Change Model

**Origin**: John Kotter, Harvard Business School, 1996

**Purpose**: Lead organizational change effectively.

### The 8 Steps

```
┌─────────────────────────────────────────────────────────────────┐
│                  KOTTER'S 8-STEP MODEL                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  CREATING THE CLIMATE FOR CHANGE                                │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 1. Create Urgency                                        │   │
│  │    - Help others see the need for change                 │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ 2. Build a Guiding Coalition                             │   │
│  │    - Assemble a group with power to lead change          │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ 3. Form a Strategic Vision                               │   │
│  │    - Create a vision to direct the change effort         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ENGAGING & ENABLING THE ORGANIZATION                           │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 4. Enlist a Volunteer Army                               │   │
│  │    - Communicate vision to get buy-in                    │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ 5. Enable Action by Removing Barriers                    │   │
│  │    - Remove obstacles, change systems that undermine     │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ 6. Generate Short-Term Wins                              │   │
│  │    - Create visible improvements, recognize contributors │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  IMPLEMENTING & SUSTAINING THE CHANGE                           │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 7. Sustain Acceleration                                  │   │
│  │    - Use wins to drive more change, don't let up         │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ 8. Institute Change                                      │   │
│  │    - Anchor new approaches in the culture                │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Step Details

#### Step 1: Create Urgency
- Identify potential threats and opportunities
- Develop scenarios of the future
- Start honest discussions about what's happening

**Key Question**: Why must we change NOW?

#### Step 2: Build a Guiding Coalition
- Identify true leaders (not just managers)
- Build a diverse, credible team
- Ensure commitment to work as a team

**Key Question**: Who has the power and influence to lead this?

#### Step 3: Form a Strategic Vision
- Determine values central to the change
- Create a short summary (1-2 sentences)
- Develop a strategy to execute the vision

**Key Question**: What will the future look like?

#### Step 4: Enlist a Volunteer Army
- Communicate the vision constantly
- Address concerns and anxieties
- Lead by example

**Key Question**: How do we get widespread buy-in?

#### Step 5: Enable Action by Removing Barriers
- Identify obstacles to change
- Remove or change systems that undermine the vision
- Recognize and reward people making change happen

**Key Question**: What's stopping people from acting?

#### Step 6: Generate Short-Term Wins
- Plan for visible performance improvements
- Create those wins
- Recognize and reward contributors

**Key Question**: How do we prove this is working?

#### Step 7: Sustain Acceleration
- Analyze what went right and wrong
- Set goals to build on momentum
- Keep ideas fresh with new change agents

**Key Question**: How do we keep momentum?

#### Step 8: Institute Change
- Articulate connections between new behaviors and success
- Develop means to ensure leadership development
- Create succession plans

**Key Question**: How do we make this stick?

## ADKAR Model

**Origin**: Prosci, 1990s

**Purpose**: Individual change management - focus on helping each person through change.

### The ADKAR Elements

```
A ──► D ──► K ──► A ──► R
│     │     │     │     │
▼     ▼     ▼     ▼     ▼
Awareness  Desire  Knowledge  Ability  Reinforcement
```

| Element | Question | Focus |
|---------|----------|-------|
| **A**wareness | Why is change needed? | Understanding the reason |
| **D**esire | What's in it for me? | Personal motivation |
| **K**nowledge | How do I change? | Training and information |
| **A**bility | Can I do it? | Skills and practice |
| **R**einforcement | Will it stick? | Sustaining the change |

### ADKAR Assessment

For each element, rate 1-5:

| Element | Score (1-5) | Barrier Point? | Action Needed |
|---------|-------------|----------------|---------------|
| Awareness | | | |
| Desire | | | |
| Knowledge | | | |
| Ability | | | |
| Reinforcement | | | |

**Barrier Point**: The first element scoring ≤3 is where to focus effort.

### ADKAR vs Kotter

| Aspect | Kotter | ADKAR |
|--------|--------|-------|
| Focus | Organizational change | Individual change |
| Approach | Top-down leadership | Person-by-person |
| Best for | Large transformations | Any change, any scale |
| Measurement | Milestone-based | Individual assessment |

**Purpose**: Assess and prioritize risks based on probability and impact.

### The Matrix

```
                          IMPACT
              Low       Medium      High      Critical
         ┌─────────┬─────────┬─────────┬─────────┐

         │    3    │    6    │    9    │   12    │

R        │    2    │    4    │    6    │    8    │
O        ├─────────┼─────────┼─────────┼─────────┤
B  Low   │   LOW   │   LOW   │  MEDIUM │   HIGH  │
         │    1    │    2    │    3    │    4    │
         └─────────┴─────────┴─────────┴─────────┘
```

### Risk Register

| Risk ID | Description | Probability | Impact | Score | Owner | Mitigation |
|---------|-------------|-------------|--------|-------|-------|------------|
| R001 | | H/M/L | H/M/L | | | |
| R002 | | H/M/L | H/M/L | | | |

### Risk Response Strategies

| Strategy | When to Use | Example |
|----------|-------------|---------|
| **Avoid** | High impact, can eliminate | Change scope to remove risk |
| **Mitigate** | Can reduce probability or impact | Add testing, redundancy |
| **Transfer** | Can shift to third party | Insurance, outsourcing |
| **Accept** | Low impact or unavoidable | Document and monitor |

**Purpose**: Translate strategy into measurable objectives across four perspectives.

```
│ │          │◄────────│     &       │────────►│ PROCESS  │      │
│ │"How do   │         │  STRATEGY   │         │          │      │

│                              │               └──────────┘      │
│                              ▼                                  │
│                    ┌─────────────────────┐                      │
│                    │ LEARNING & GROWTH   │                      │

```

### Perspective Details

| Perspective | Focus | Example Measures |
|-------------|-------|------------------|
| **Financial** | Shareholder value | Revenue growth, profit margin, ROI, EVA |
| **Customer** | Customer satisfaction | NPS, retention rate, market share, satisfaction |
| **Internal Process** | Operational excellence | Cycle time, quality, productivity, cost per unit |
| **Learning & Growth** | Capabilities | Employee engagement, skills, innovation, IT systems |

### Balanced Scorecard Template

| Perspective | Objective | Measure | Target | Initiative |
|-------------|-----------|---------|--------|------------|
| **Financial** | Increase profitability | Operating margin | 15% | Cost reduction program |
| **Financial** | Grow revenue | Revenue growth | 10% YoY | New product launch |
| **Customer** | Improve satisfaction | NPS | >50 | Customer service training |
| **Customer** | Increase loyalty | Retention rate | 90% | Loyalty program |
| **Internal** | Reduce cycle time | Order-to-delivery | <3 days | Process automation |
| **Internal** | Improve quality | Defect rate | <1% | Six Sigma program |
| **Learning** | Develop skills | Training hours | 40/employee | Learning platform |
| **Learning** | Increase engagement | Engagement score | >80% | Culture initiative |

```
                 │               │
           Revenue Growth   Productivity
                 ↑               ↑
                 │               │

            Acquisition     Retention
                 ↑               ↑

            Development     Excellence
                 ↑               ↑
```

### Implementation Steps

1. **Clarify strategy** - What are we trying to achieve?
2. **Identify objectives** - 3-5 per perspective
3. **Select measures** - Lead and lag indicators
4. **Set targets** - Ambitious but achievable
5. **Define initiatives** - Projects to achieve targets
6. **Cascade** - Deploy to departments/individuals
7. **Review & adapt** - Regular performance reviews

## OKRs (Objectives and Key Results)

**Origin**: Andy Grove at Intel, popularized by John Doerr at Google

**Purpose**: Set and track ambitious goals with measurable outcomes.

### OKR Structure

```
OBJECTIVE: What do we want to achieve? (Qualitative, inspiring)
│
├── KEY RESULT 1: How do we measure success? (Quantitative)
├── KEY RESULT 2: How do we measure success? (Quantitative)
└── KEY RESULT 3: How do we measure success? (Quantitative)
```

### Characteristics

| Component | Characteristics |
|-----------|----------------|
| **Objective** | Qualitative, inspiring, time-bound, actionable |
| **Key Results** | Quantitative, measurable, achievable, leads to objective |

### OKR Examples

**Company Level**:
```
Objective: Become the market leader in customer satisfaction

KR1: Increase NPS from 40 to 60
KR2: Reduce customer complaint resolution time from 48h to 12h
KR3: Achieve 95% customer retention rate
```

**Team Level**:
```
Objective: Launch a world-class mobile app

KR1: Achieve 4.5+ star rating in app stores
KR2: Reach 100,000 monthly active users
KR3: Achieve <2 second load time on all screens
```

**Individual Level**:
```
Objective: Become an expert in data analytics

KR1: Complete advanced SQL certification
KR2: Build 5 self-service dashboards for the team
KR3: Train 10 colleagues on basic data analysis
```

### OKR Best Practices

| Do | Don't |
|----|-------|
| Set 3-5 objectives max | Set too many objectives |
| Make OKRs ambitious (70% achievement is good) | Make OKRs sandbagged |
| Focus on outcomes, not tasks | List activities as key results |
| Review weekly | Set and forget |
| Keep key results measurable | Use vague language |
| Align across teams | Create OKRs in silos |

### OKR Cadence

| Timeframe | Purpose |
|-----------|---------|
| **Annual** | Company vision and strategic OKRs |
| **Quarterly** | Team and department OKRs |
| **Weekly** | Check-in on progress |
| **End of Quarter** | Score and reflect |

### OKR Scoring

| Score | Meaning |
|-------|---------|
| 0.0 - 0.3 | Failed to make progress |
| 0.4 - 0.6 | Made progress but fell short |
| 0.7 - 1.0 | Achieved the goal (0.7 = stretch goal achieved) |

### OKRs vs KPIs vs Balanced Scorecard

| Aspect | OKRs | KPIs | Balanced Scorecard |
|--------|------|------|-------------------|
| Purpose | Drive change, set ambitious goals | Monitor performance | Strategic alignment |
| Timeframe | Quarterly | Ongoing | Annual/ongoing |
| Scope | Focused priorities | Comprehensive metrics | Four perspectives |
| Achievement | 70% is success | 100% is expected | Varies by target |
| Best for | Startups, innovation, change | Operations, stability | Large organizations |

## McKinsey Horizon Model

**Origin**: McKinsey & Company

**Purpose**: Balance short-term performance with long-term growth.

### The Three Horizons

```
PROFIT/
GROWTH
   │
   │                                          ╱
   │                                        ╱  Horizon 3
   │                                      ╱    (Create viable options)
   │                               ╱────╱
   │                         ╱────╱
   │                   ╱────╱         Horizon 2
   │             ╱────╱               (Build emerging businesses)
   │       ╱────╱
   │ ╱────╱            Horizon 1
   │╱                  (Extend core business)
   └──────────────────────────────────────────── TIME
        Now      Near-term    Medium    Long-term
```

### Horizon Details

| Horizon | Focus | Timeframe | Metrics | Management Style |
|---------|-------|-----------|---------|-----------------|
| **H1** | Core business | 0-1 years | Revenue, profit, market share | Operational excellence |
| **H2** | Emerging businesses | 1-3 years | Growth rate, customers, capability | Venture development |
| **H3** | Future options | 3-5+ years | Options created, experiments run | Exploration, R&D |

### Portfolio Allocation

Typical resource allocation:
- **Horizon 1**: 70% of resources
- **Horizon 2**: 20% of resources
- **Horizon 3**: 10% of resources

### Application

| Question | Framework Response |
|----------|-------------------|
| "How do we balance today vs. tomorrow?" | Use horizons to categorize initiatives |
| "Where should we invest?" | Ensure coverage across all three horizons |
| "How do we measure innovation?" | Different metrics for each horizon |
| "How do we manage the portfolio?" | Different governance for each horizon |
